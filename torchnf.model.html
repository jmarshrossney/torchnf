inference_step(data: Tensor) → tuple[torch.Tensor, torch.Tensor][source]

Encodes the data and computes the density under the model.

Encoding step:

First, encode() is called to transform the data \(x\) to \(z = \mathrm{encode}(x)\), and compute the log Jacobian determinant \(\log \lvert \partial z / \partial x \rvert\) of the transformation.

Density estimation:

Computes an estimate of the log probability density of the input data.

This estimate is the log probability density of the data under the model, \(q(x)\). That is, the push-forward of \(q(z)\) by the decoding transformation. Since the encoder/decoder is invertible, this can be calculated as follows:

\[\log q(x) = \log q(z) + \log \left\lvert \frac{\partial z}{\partial x} \right\rvert\]

Log likelihood:

The log-likelihood of the input data with respect to the model is given by

\[\log \ell(\{x\}) = \frac{1}{n} \sum_{\{x\}} - \log q(x) \, .\]
Parameters

data – the input data, \(x\)

Returns

  • sample – the encoded sample, \(z\)

  • log_prob – the log probability density, \(\log q(x)\)

  • loss – the negative log-likelihood, \(\log \ell(\{x\})\)

Return type

sampling_step(noise: Tensor) → dict[str, torch.Tensor][source]

Decodes the latents and computes the density under the model.

Decoding step:

First, decode() is called to transform the latent variables \(z\) to \(x = \mathrm{decode}(z)\), and compute the log Jacobian determinant of the transformation, \(\log\lvert\partial x/\partial z\rvert\)

Push-forward:

The probability density at point \(x = \mathrm{decode}(z)\) is the push-forward of \(q(z)\) by the decoding transform:

\[\log q(x) = \log q(z) - \log \left\lvert \frac{\partial x}{\partial z} \right\rvert\]
