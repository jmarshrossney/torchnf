torchnf.utils.expand_elements(x: Tensor, data_shape: Size, stack_dim: int = 0) → Tensor

Expands and stacks each element of a one-dimensional tensor.

The input tensor is split into chunks of size 1 along the zeroth dimension, each element expanded, and then the result stacked.

Parameters

  • x – One-dimensional input tensor

  • data_shape – Shape that each element will be expanded to

  • stack_dim – Dimension that the resulting expanded elements will be stacked on

Effectively this does the following:

elements = x.split(1)
elements = [el.expand(data_shape) for el in elements]
out = torch.stack(elements, dim=stack_dim)
return out
torchnf.utils.metropolis_test(log_weights: Tensor) → list

Subjects a set of log-weights to the Metropolis test.

The Metropolis-Hastings algorithm generates an asymptotically unbiased sample from some ‘target’ distribution \(p(y)\) given a proposal distribution \(q(y)\) which can be sampled from exactly.

For each member of the sample, \(y\), the log-weight is defined as

\[\log w(y) = \log p(y) - \log q(y)\]

A Markov chain is constructed by running through the sample sequentially and transitioning to a new state with probability

\[A(y \to y^\prime) = \min \left( 1, \frac{q(y)}{p(y)} \frac{p(y^\prime)}{q(y^\prime)} \right) \, .\]
Parameters

log_weights – One-dimensional tensor containing N log weights

Returns

A set of N-1 indices corresponding to the state of the Markov chain at each step.

Note

Note that the Markov chain is initialised using the first element.
